System Pipeline
Processing Pipeline
END-TO-END DATA FLOW
01
Data Ingestion
Restaurant surplus records, shelter demand profiles, and global geographic coordinates ingest from FAO, WFP, and city datasets.
INPUT
02
Cleaning & Feature Engineering
Missing value imputation, normalization, and derived features: distance matrices, surplus averages, demand volatility indices.
ETL
03
K-Means Clustering
Unsupervised segmentation of restaurants into high / medium / low surplus zones. DBSCAN as an alternative for density-based anomalies.
ML
04
Demand Prediction
Supervised models (Random Forest, XGBoost, Neural Networks) forecast daily & weekly shelter demand and urgency scores.
ML
05
Graph Construction
Bipartite graph G = (V, E) built with restaurants and shelters as nodes; edge weights encode distance, surplus, and urgency.
GRAPH
06
Max-Flow Optimization
Ford-Fulkerson & Dinic's algorithm compute optimal food distribution routes. Hopcroft-Karp handles large-scale bipartite matching.
OPTIMIZE
07
Distribution Plan + Evaluation
Output: matched routes, meal counts, RMSE / F1 / Precision metrics, and a real-time dashboard visualization.
OUTPUT
Mathematical Model
OPTIMIZATION OBJECTIVE
G = (V, E)
// V = restaurants ∪ shelters

Maximize  Σ flow(r, s)

subject to:
  flow(r,s) ≤ surplus(r)
  flow(r,s) ≤ demand(s)
  dist(r,s) ≤ D_max

// Capacity: C(r,s) = min(surplus, demand)

Graph Optimization Algorithms
Ford–Fulkerson
O(E · max_flow)
Finds maximum food flow from restaurant sources to shelter sinks by iteratively augmenting along paths with remaining capacity.
Augmenting path 1
Augmenting path 2
flow / capacity
Hopcroft–Karp
O(E √V)
Efficient bipartite matching. Finds maximum cardinality pairings between restaurants and shelters using BFS phase + DFS augmentation.
Matched pairs
Candidate edges
Restaurant nodes
Dinic's Algorithm
O(V² · E)
Builds a layered level graph via BFS, then pushes blocking flows via DFS. Dramatically faster than Ford-Fulkerson on large dense networks.

L2 intermediate relay
